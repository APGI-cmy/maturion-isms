## Defects Closed

### S-032 — CI Token File Pattern Mismatch — REMEDIATED

**Problem**: The `iaa-assurance-check` job in `agent-contract-audit.yml` searched for
`assurance-token-*.md` in `.agent-admin/assurance/`, but the IAA writes `iaa-token-session-*.md`.
CI was unable to automatically confirm IAA token presence.

**Fix**: `.github/workflows/agent-contract-audit.yml` Check 1 updated to search for both:
- `iaa-token-session-*.md` (canonical IAA output format)
- `assurance-token-*.md` (legacy format, retained for backward compatibility)

**Evidence**: `find "$ASSURANCE_DIR" \( -name "iaa-token-session-*.md" -o -name "assurance-token-*.md" \)`
**File changed**: `.github/workflows/agent-contract-audit.yml`

### S-033 — OVL-CI-005 "Inherent Limitation Exception" Documentation — REMEDIATED

**Problem**: OVL-CI-005 required CI run evidence before IAA handover, but for self-referential
workflow PRs (workflows that only fire post-merge), this is physically impossible to satisfy
before merge.

**Fix**: `.agent-workspace/independent-assurance-agent/knowledge/iaa-category-overlays.md`
updated to v3.3.0 with:
- OVL-CI-005 table entry extended with explicit Inherent Limitation Exception clause
- New "OVL-CI-005 Inherent Limitation Exception — Detailed Guidance" section defining:
  - What constitutes a self-referential workflow PR
  - Why the exception is necessary
  - Three required substitutes (YAML validation, pattern parity, workflow_dispatch retention)
  - Retroactive incident acceptance policy
- IAA knowledge index updated to v2.8.0

**File changed**: `.agent-workspace/independent-assurance-agent/knowledge/iaa-category-overlays.md`
**File changed**: `.agent-workspace/independent-assurance-agent/knowledge/index.md`

### S-007 — POLC Boundary Gate Machine Enforcement (Foreman Implementation Check) — REMEDIATED

**Problem**: The `polc-boundary-gate.yml` had a single job named
`"Merge Gate Interface / polc-boundary/validation"` which did not match the
`merge_gate_interface.required_checks` names in the Foreman v2 agent contract.

**Fix**: `.github/workflows/polc-boundary-gate.yml` refactored:
- Workflow renamed from "POLC Boundary Validation Gate" to "POLC Boundary Validation"
- Split into 3 separate named jobs matching required_checks:
  - `foreman-implementation-check` — detects Foreman implementation commits (HARD FAIL)
  - `builder-involvement-check` — validates builder delegation + pre-brief (includes S-023)
  - `session-memory-check` — validates session memory presence and compliance

**File changed**: `.github/workflows/polc-boundary-gate.yml`

---

### S-023 — Pre-Brief Existence CI Gate — REMEDIATED

**Problem**: No machine-enforced CI gate existed to fail a PR when implementation file changes
were present but no `.agent-admin/assurance/iaa-prebrief-*.md` artifact existed. This allowed
retroactive pre-brief commits after implementation work began.

**Fix**: Added to the `builder-involvement-check` job in `polc-boundary-gate.yml`:
- Hard gate check: `find "$ASSURANCE_DIR" -name "iaa-prebrief-*.md"` — fails with `exit 1` if absent
- Clear diagnostic output identifying the A-031 violation (PRE-BRIEF-BEFORE-DELEGATION)
- Instructions for remediation (commit wave-current-tasks.md, await pre-brief, then delegate)

**File changed**: `.github/workflows/polc-boundary-gate.yml`
