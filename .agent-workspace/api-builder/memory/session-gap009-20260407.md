## Architecture Decisions

### SupabaseMinimalClient Interface
Used the same minimal-interface pattern as `SupabasePersistentMemoryAdapter.ts` — avoids
`@supabase/supabase-js` as a direct dependency of `@maturion/ai-centre`. Interface is
structurally typed: any compatible client (test mock or real Supabase JS client) satisfies it.

Added `rpc()` method to the interface to support the `set_config` call for RLS enforcement.

### RLS Enforcement Pattern
Following SB-003: before every INSERT, call:
```typescript
await this.supabase.rpc('set_config', {
  parameter: 'app.current_organisation_id',
  value: entry.organisationId,
  is_local: true,
});
```
This sets the Postgres config variable consumed by the `ai_episodic_events_insert_org_scope`
RLS policy.

### Constructor Guard
Changed from `supabaseClient?: any` with `arguments.length > 0` check to `supabaseClient: SupabaseMinimalClient`
(typed, required). The falsy guard `if (!supabaseClient)` covers both `undefined` and `null`.
All existing tests asserting the constructor throw remain GREEN.

### Error Handling (SB-007)
Both `record()` and `retrieve()` check the `{ error }` return from Supabase and throw
immediately if set. No silent swallowing.

### Row Mapping (SB-002)
`toRow()` maps all required fields: `organisation_id`, `session_id`, `agent_id`, `event_type`,
`capability`, `summary`, `full_context`. The `capability` field maps from `EpisodicEventEntry.capability`
which is typed as `Capability` enum — the enum values match the CHECK constraint in the migration.

`fromRow()` reverses the mapping, handling nullable `full_context` and optional `created_at`.

## BD-000 User Journey Declaration

**Write path**:
> "Agent invokes `EpisodicMemoryAdapter.record(entry)` →
> adapter calls `rpc('set_config', { parameter: 'app.current_organisation_id', value: entry.organisationId, is_local: true })` →
> adapter executes `from('ai_episodic_events').insert(toRow(entry))` with all required fields →
> Supabase enforces RLS policy (organisation_id matches current_setting) →
> row persisted to database →
> adapter returns without error."

**Read path**:
> "Caller invokes `EpisodicMemoryAdapter.retrieve({ organisationId, sessionId?, limit? })` →
> adapter executes `from('ai_episodic_events').select('*').eq('organisation_id', organisationId)[.eq('session_id', sessionId)].order('created_at', { ascending: false })` →
> Supabase enforces RLS for row-level isolation →
> rows mapped via `fromRow()` from snake_case to camelCase →
> optional limit applied →
> filtered results returned as `EpisodicEventEntry[]`."

**Error path**:
> "INSERT fails (DB error or RLS violation) →
> adapter receives `{ error }` from Supabase →
> adapter throws the error →
> caller receives rejection (no silent swallowing of Supabase errors)."
