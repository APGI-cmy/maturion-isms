## Lessons / What Future Sessions Should Know

1. **`organisation_settings.id` IS the organisation FK** — the table uses `id UUID PRIMARY KEY REFERENCES public.organisations(id)`, meaning `id = organisation_id`. RLS policies must use `id IN (SELECT organisation_id FROM profiles WHERE ...)` not `organisation_id IN (...)`.

2. **mini_performance_standards is service_role write-only** — NEVER add INSERT/UPDATE policies for this table. T-PBF2-008 is a permanent security guard. Any INSERT/UPDATE policy on this table will cause T-PBF2-008 to fail.

3. **evidence DELETE is always required with INSERT+UPDATE** — if you see evidence in a gap register, assume DELETE is also required unless explicitly stated otherwise. The IAA pre-brief explicitly flags missing DELETE as CFM.

4. **File-based tests allow instant local verification** — these migration tests read the SQL file directly; no Supabase environment needed. `npx vitest run modules/mat/tests/security-rls/wave-postbuild-fails-02.test.ts` runs in ~330ms.
