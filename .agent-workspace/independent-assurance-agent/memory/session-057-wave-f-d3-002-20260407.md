## Learning Notes

1. **Security remediation PRs benefit from explicit threat-model tables in pre-brief**: The
   pre-brief §3 threat model (Before/After table) made the journey trace in BD-000 efficient
   and unambiguous. This pattern should be encouraged for future security remediations.

2. **Factory-not-called assertion is a strong security test pattern**: W9.4-T-011 asserts
   `expect(mockFactory).not.toHaveBeenCalled()` to verify the pipeline is never instantiated
   when auth fails. This pattern provides stronger security guarantee than merely checking the
   response status code — it proves the sensitive operation was never triggered. IAA should
   look for this pattern in future security-gating test reviews and flag its absence.

3. **Test fixture `'secret-arc-token'` pattern is correct and not a BD-016 violation**: When
   a test sets `process.env['ARC_APPROVAL_TOKEN'] = 'secret-arc-token'` with `afterEach` cleanup,
   this is a standard test harness fixture, not a hardcoded production secret. Production code
   has no hardcoded fallback. Future IAA sessions should distinguish test-only env fixtures from
   hardcoded production secrets.

4. **BLOCKER-01 pending.ts deferral pattern**: When a PR correctly scopes out a related finding
   with a clear Option A declaration, IAA should accept the deferral and confirm it rather than
   treating it as a scope gap. The key requirement is that the deferral is explicitly declared,
   not silent.
