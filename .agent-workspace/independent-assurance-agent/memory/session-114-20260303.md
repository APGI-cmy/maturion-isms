## Learning Notes

1. **Three-invocation resolution pattern observed**: This is the first triple-invocation resolution cycle in IAA's operational history. Session-112 identified 5 root-cause failures; session-113 caught 1 regression introduced during the rewrite; session-114 (this session) confirmed all 6 resolved. The pattern confirms: IAA's STOP-AND-FIX mandate works. Each invocation isolated exactly the remaining failures. The zero-severity-tolerance rule (CORE-021) is essential — it forced the OVL-AC-011 fix even when it was a "small" regression.

2. **PREHANDOVER rewrite regression risk confirmed as real**: Session-113 predicted this pattern. The fix (eac1c79) was precise and correct — adding only the "Chars Before" column. CodexAdvisor treated the rewrite as additive (a single column addition) rather than a full document replacement. This avoided introducing new regressions. The A-026 candidate (treat PREHANDOVER as additive, not replacement) observed in session-113 remains a valuable potential rule.

3. **Per-agent bash template scoping is correct architecture**: The decision to scope `find .agent-workspace/<agent-name> -name 'blocker-*.md'` per-agent (rather than the previous broad `.agent-workspace/` scan) correctly isolates each builder's responsibility. The session-112 false claim of 0 blockers (when there were 4 in other agents' workspaces) is now architecturally impossible with per-agent scoping.

4. **policy_refs YAML structure note**: `policy_refs` is a TOP-LEVEL YAML key in builder contracts — NOT nested under `governance`. This is important for future automated checks. Manual YAML parse must check `data.get('policy_refs')`, not `data.get('governance',{}).get('policy_refs')`.
