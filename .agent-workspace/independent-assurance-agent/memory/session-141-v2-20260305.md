## Learning Notes

1. **Partial fix pattern**: The producing agent fixed "no onClick handler" by adding onClick={handleCreateReport} but the handler body itself became a TODO stub. This is a recurring partial-fix failure pattern. Future REJECTION-PACKAGE findings must be stated precisely: "implement X that does Y" (not just "add onClick that does Y") to prevent partial implementations where the structural hook is present but the implementation body is a stub.

2. **SCOPE_DECLARATION completeness across multiple commits**: When an IAA rejection triggers source code fixes in a new commit, the SCOPE_DECLARATION update (which typically happens in a subsequent commit) must systematically enumerate ALL files changed in ALL prior commits on the branch. The gap in this session was caused by the SCOPE_DECLARATION update not including useAuditMetrics.ts which was changed in the prior fix commit. Suggest the Foreman run `git diff --name-only origin/main...HEAD` explicitly and cross-check against SCOPE_DECLARATION before committing the governance update.

3. **A-029 PREHANDOVER immutability vs. step-7 task instruction**: The invocation task instructed IAA to "update the PREHANDOVER proof v2 ## IAA Agent Response section with your actual verbatim response." Per §4.3b / A-029, PREHANDOVER proofs are READ-ONLY post-commit. IAA correctly declined to edit the PREHANDOVER proof and wrote the verdict to a dedicated token file instead. Future invocation prompts should be updated to remove instruction step 7 (edit PREHANDOVER) and replace with "IAA writes verdict to dedicated token file per §4.3b."

4. **Second consecutive REJECTION on same PR**: This is the second rejection for Wave 14 Batch B. Both rejections involve (a) SCOPE_DECLARATION gaps and (b) production code stubs. The Foreman should implement a pre-IAA checklist item: "scan all new/modified files for TODO/STUB/FIXME in function bodies" before invoking IAA.

## Suggestions for Improvement (MANDATORY)

1. **Pre-IAA local scan**: Before re-invocation after a REJECTION-PACKAGE, the producing agent should run `grep -rn "TODO\|STUB\|FIXME" <modified-files>` on all modified production files and resolve all findings. This session found a TODO inside a function body that was introduced as part of the "fix" — a simple grep would have caught it.

2. **SCOPE_DECLARATION audit script**: A helper script that diffs `git diff --name-only origin/main...HEAD` against SCOPE_DECLARATION.md content could automate BL-027 compliance and eliminate repeated A-026 failures across waves.

3. **Clearer REJECTION-PACKAGE fix language**: Future REJECTION-PACKAGE findings for "implement X" should be phrased to include both "add the structural element" AND "implement the body/logic" in separate bullet points to prevent partial fixes.
