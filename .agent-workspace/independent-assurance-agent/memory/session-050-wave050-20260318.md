## Learning Notes

1. **OVL-CI-005 self-referential exception**: This is the second PR in recent sessions where
   `pull_request_target` workflows were modified without explicitly invoking OVL-CI-005.
   The `workflow_dispatch` requirement in particular is easy to forget. Consider adding a
   FAIL-ONLY-ONCE rule that any modified `pull_request_target` workflow that adds new jobs
   must include `workflow_dispatch:` unless explicitly waived by CS2.

2. **CANON_INVENTORY as a mandatory post-update step**: CANON_INVENTORY stale hash is a
   recurring pattern (wave-18 also had this). CodexAdvisor should add CANON_INVENTORY update
   to its standard canon-change checklist. The QP gate should include a CANON_INVENTORY hash
   parity check.

3. **Knowledge index drift**: Tier 2 knowledge files updated without corresponding index.md
   updates are a recurring gap. The prehandover-template PREHANDOVER checklist should include
   an explicit "did you update index.md?" checkpoint.

4. **Ripple completeness for canon-to-contract chains**: When canon introduces new output
   requirements on an agent (PHASE_B_BLOCKING_TOKEN on IAA), the ripple must follow to that
   agent's contract. CodexAdvisor's ripple assessment should include a check: "does this canon
   change require any agent contract to specify new output fields?"
