1. **A-021 meta-pattern: remediation applied but not committed.** The foreman executed all three session-133 fix instructions in the working tree but did not commit before invoking IAA. This is a workflow gap: the foreman's pre-invocation checklist must include `git status` to confirm a clean working tree. 2. **A-029 conflict with session-133 fix instruction.** Session-133's FAILURE-2 fix said "add iaa_audit_token field to proof; replace PHASE_A_ADVISORY text." 5. **A-021 pre-check proposal (potential A-031):** Before any re-invocation, the invoking agent should run `git status` and confirm output is "nothing to commit, working tree clean" (or only IAA-output files staged for the current session). This would catch the working-tree-only remediation pattern immediately and avoid IAA consuming a full session to re-reject on the same root cause.
