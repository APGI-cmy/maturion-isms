## Learning Notes

1. **Missing iaa_audit_token is a ceremony preparation failure, not a post-IAA failure**: The producing agent must add `iaa_audit_token: PENDING` and `## IAA Agent Response (verbatim)` section to the PREHANDOVER proof BEFORE invoking IAA. This is the mandatory pre-invocation state. The absence of these fields (rather than having them as PENDING) means the PREHANDOVER proof was created without following the ceremony template. Future governance-liaison-isms sessions should consult the session-memory-template.md and CORE-018/CORE-016 detail before creating the PREHANDOVER proof.

2. **OVL-KG-003 (version history tables) is a structural gap in governance-liaison-isms Tier 2 knowledge files**: Unlike IAA's own Tier 2 files (iaa-core-invariants-checklist.md etc.) which have `## Version History` tables, governance-liaison-isms's Tier 2 files track version only in header metadata. OVL-KG-003 requires structured version history tables. Governance-liaison-isms needs to add these tables to all three knowledge files before future KNOWLEDGE_GOVERNANCE PRs will pass.

3. **OVL-KG-004 (stale index.md) is a recurring pattern across multiple PRs**: This is the second session where index.md stale version entries were found. The self-referential entry (`index.md (this file)`) is particularly prone to being forgotten when the knowledge version bumps. Producing agents should verify the index.md table lists ALL file versions in sync with the actual files on the branch.

4. **A-09 vs A-009 padding inconsistency was acknowledged but not committed**: The PR description lists "A-09 ID standardized to A-009" as a post-commit fix, but the committed FAIL-ONLY-ONCE.md at HEAD still shows `A-09`. This suggests the fix was made to the PR description or notes but the actual file was not updated. Producing agents must commit all fixes to the branch before invoking IAA (per A-021) — written descriptions of fixes that aren't in the committed files are not evidence of fix completion.

5. **First invocation for governance-liaison-isms KNOWLEDGE_GOVERNANCE PRs reveals structural gaps**: This is likely the first KNOWLEDGE_GOVERNANCE-category IAA invocation for governance-liaison-isms artifacts. The structural gaps (no version history tables, missing ceremony fields) suggest the producing agent may not have previously reviewed the OVL-KG overlay requirements before preparing the PREHANDOVER proof. Future invocations will be cleaner once these structural elements are established.
