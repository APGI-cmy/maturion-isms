## Learning Notes

1. **BL-027 format dependency is invisible until script runs**: The SCOPE_DECLARATION format issue (table vs list) is a silent failure — the foreman can write a correct-looking Wave 14 section in table format, reference the right 14 files, and still have BL-027 fail completely because the parsing regex doesn't support table format. The foreman should always reference the Wave 13 B+C section (which passed BL-027) as the format template and match it exactly.

2. **Prior-wave SCOPE_DECLARATION entries must be trimmed**: When the SCOPE_DECLARATION was created in Wave 13 B+C and merged, those 18 list-format entries went into origin/main. When this PR (Wave 14) runs, git diff shows only the new 21 files — not the 18 already-merged files. But SCOPE_DECLARATION still has those 18. This creates 18 "EXTRA" declarations in BL-027's set comparison. The foreman needs to trim the SCOPE_DECLARATION to contain ONLY the current PR's files before IAA invocation.

3. **Very close to ASSURANCE-TOKEN**: This is the fourth IAA invocation on this PR. The single remaining issue is a mechanical SCOPE_DECLARATION format/content fix. All implementation, all governance content, all other ceremony artifacts are confirmed PASS. The fix is minimal: rewrite Wave 14 SCOPE_DECLARATION section in list format with all 21 files and remove prior-wave entries.
