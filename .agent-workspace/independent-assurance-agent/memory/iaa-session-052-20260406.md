## Suggestions for Improvement

**MANDATORY FIELD — Concrete improvement observation from this session:**

1. **Pre-brief scope completeness for multi-issue branches**: When Foreman bundles multiple issues
   onto a single branch (e.g., issue #1253 + issue #1255 on `copilot/align-live-maturation-artifacts`),
   the IAA Pre-Brief must enumerate ALL qualifying tasks across ALL bundled issues. In this session,
   the pre-brief covered #1255 BUILD_PROGRESS_TRACKER tasks but explicitly classified
   AGENT_CONTRACT as "Not applicable" — which was incorrect for the #1253 foreman contract change.
   Suggested addition to Pre-Brief protocol: Foreman should scan for `.github/agents/` changes
   when generating the pre-brief, regardless of which issue triggered the scan.

2. **Ripple assessment CodexAdvisor self-check**: The ripple assessment absence (A-023) has
   recurred multiple times. IAA recommends that CodexAdvisor add a standing self-check to its
   PREHANDOVER generation template: "Have I written `## Ripple/Cross-Agent Assessment`? YES/NO."
   This would eliminate the most common single cause of REJECTION-PACKAGE in overseer-class PRs.
