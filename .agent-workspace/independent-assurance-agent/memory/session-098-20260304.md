## Learning Notes
1. **Policy count propagation error**: When evidence documents say "N total" but the deliverable has N+1, the error propagates across all referencing documents (SCOPE_DECLARATION, wave-current-tasks.md, multiple PREHANDOVER sections). Each reference must be independently verified against the actual artifact. New pattern: during OVL-AM-002 review, IAA should ALWAYS count implementation items (policies, tests, etc.) directly from the artifact rather than accepting the stated count in documentation.

2. **CORE-016/CORE-018 vs A-029 design gap**: CORE-018 condition 4 requires `## IAA Agent Response (verbatim)` in PREHANDOVER proof, but A-029 makes proof immutable post-commit. For first-time invocations, satisfying condition 4 is architecturally impossible. The spirit of CORE-016 is INC-IAA-SKIP-001 detection (did IAA actually run?). When IAA IS running legitimately, the check is satisfied by the existence of the dedicated verdict token file (§4.2b). Recommend: Update CORE-018 to add a first-invocation carve-out for A-029-compliant proofs.

3. **Implementation plan path as static planning artifact**: implementation-plan.md records the PLAN at definition time. When builders land files at paths different from the plan, the implementation plan is not automatically updated. IAA should verify test/artifact paths against actual filesystem, not just accept plan paths as ground truth.
