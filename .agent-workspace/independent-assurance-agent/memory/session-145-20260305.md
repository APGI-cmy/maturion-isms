## fail_only_once_updates

New pattern observed: The producing agent has now failed A-026 across two consecutive re-invocations on the same PR — different files each time:
- Session 143: SCOPE_DECLARATION had session-047 content entirely (wrong session)
- Session 145: SCOPE_DECLARATION updated for session-048 but missed IAA session-144 artifacts added between FFA fixes and SCOPE update

Root cause: When IAA adds files to a branch (via REJECTION-PACKAGE commits), those files become part of the PR diff. The producing agent must include ALL branch diff files in SCOPE_DECLARATION — including IAA-written files added by prior rejection process. The producing agent included session-143 IAA files but missed session-144 IAA files, suggesting the SCOPE_DECLARATION update was partially based on session-143 content and not a fresh `git diff --name-only` audit.

**Recommendation for new FAIL-ONLY-ONCE rule**: When updating SCOPE_DECLARATION.md on a re-invocation, producing agent MUST run `git diff --name-only origin/main...HEAD` (or equivalent) and compare the output against the existing SCOPE_DECLARATION BEFORE committing. This is the only reliable way to ensure exact match. The pattern of "include what I remember" instead of "include what git says" is the root failure.

This is not yet at the A-027 threshold (same PR, three consecutive) but is session 2 of a potential pattern. Recording for escalation if session 145's fix also misses files.
