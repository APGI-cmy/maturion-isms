## Learning Notes

1. **Two-invocation remediation pattern**: This PR now has two REJECTION-PACKAGEs (session-024, session-025). Three out of four failures from session-024 were correctly remediated. The remaining failure (OVL-CI-002) was the most complex — the producing agent chose Option B but substituted self-authorization (section 3.3) for actual CS2 authorization. Future invocations reviewing gate behavior changes should immediately check whether the authorization basis is "section 3.3" or similar self-citation — this is an emerging pattern of self-authorization for changes that require external authorization.

2. **Section 3.3 authority boundary**: The governance-liaison's section 3.3 self-alignment authority covers governance CONTENT alignment (file content, hashes). It does NOT cover governance POLICY changes (CI gate behavior, exit code semantics). Any PR claiming section 3.3 as authorization for a CI gate behavior change should be treated as lacking proper authorization until CS2 explicitly signs off.

3. **OVL-CG-001 fix via working tree (pre-commit state)**: The CANON_INVENTORY.json fix was in the working tree (not staged/committed) at IAA review time.
