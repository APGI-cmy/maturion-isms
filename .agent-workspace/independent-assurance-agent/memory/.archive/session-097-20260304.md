## learning_notes
1. Pattern identified: "Post-build, pre-handover invocation" — Foreman submitted work for IAA audit
   BEFORE creating the PREHANDOVER proof, framing it as an optional pre-check. This is BACKWARD
   from the governance ceremony: PREHANDOVER proof must be committed BEFORE IAA invocation
   (CORE-018, A-021). The pre-handover framing does not create a governance carve-out.
2. POLC breach disclosed but work is technically sound: When a POLC breach is disclosed and
   builders independently verify the work as correct, IAA does not need to re-audit technical
   content from scratch — but the governance ceremony (PREHANDOVER proof, session memory,
   SCOPE_DECLARATION) still must be complete. Technical correctness does not substitute for
   ceremony compliance.
3. SCOPE_DECLARATION.md stale pattern (A-026): Seen in sessions 124/125/126/127 (governance
   ripple PR) and now again here. The Foreman's workflow must include SCOPE_DECLARATION.md
   update as a mandatory step before IAA invocation. Adding to PREHANDOVER template as standing
   requirement would prevent recurrence.
4. supabase-sync-audit document pattern: The Foreman committed an excellent `supabase-sync-audit-20260304.md`
   that contains most of the OVL-AM-004/005/008 content. The only missing piece is referencing
   it formally in the PREHANDOVER proof. This is a quick fix — the substantive work was done.

## Suggestions for Improvement
1. CONCRETE IMPROVEMENT: Add a "pre-IAA checklist" to the Foreman PREHANDOVER template that
   explicitly lists: (a) PREHANDOVER proof committed, (b) session memory committed, (c)
   SCOPE_DECLARATION.md updated and matches diff, (d) all overlay sections populated. This would
   have prevented all 9 failures in this session.
2. CEREMONY SEQUENCING NOTE: The distinction between "post-build" and "pre-handover" should be
   clarified in Foreman documentation — IAA invocation IS part of the handover ceremony, not a
   step before it begins. The PREHANDOVER proof is a pre-condition for IAA, not a post-condition.
