## Suggestions for Improvement (MANDATORY)

1. **Compound session sub-identifiers**: IAA sessions that audit multiple PRs in sequence should issue per-PR sub-tokens (`IAA-session-NNNa-PASS`, `IAA-session-NNNb-PASS`) rather than a single shared token. This closes the A-016 grey area for compound sessions and makes token-to-PR traceability unambiguous. Candidate for Foreman FAIL-ONLY-ONCE S-010.

2. **Audit task vs. merge gate invocation distinction**: The current IAA session memory template presupposes a PR merge gate invocation (`pr_reviewed`, `verdict: ASSURANCE-TOKEN / REJECTION-PACKAGE`). Audit-only tasks (like CL-4-D4) don't fit this model cleanly. A supplementary template variant for audit-task sessions would improve session memory quality and make the distinction explicit.
