**Check**: §4.3 governance/alignment + FAIL-ONLY-ONCE A-028
**Evidence**: `bash .github/scripts/validate-scope-to-diff.sh` exits 1:
  "SCOPE_DECLARATION.md is empty or malformed — 0 declared files vs 15 in diff"
  
The SCOPE_DECLARATION.md content is CORRECT (all 15 files identified and listed) but FORMAT is WRONG.
File declaration lines use em-dash (`—`, U+2014) as separator:
  `- \`file\` — description`
BL-027 requires hyphen (`-`, U+002D) as separator:
  `- \`file\` - description`
The BL-027 regex `^\s*-\s+\`[^`]+\`\s+-\s+` does not match em-dash. The fallback regex also fails 
(it excludes backtick-prefixed paths). Result: 0 declared files detected → validation fails.

**Fix required**: Replace all em-dash (`—`) separators with hyphen-space (`- `) in the file declarations 
section of SCOPE_DECLARATION.md. Each line should be:
  `- \`path/to/file\` - one-line description`
NOT:
  `- \`path/to/file\` — one-line description`
Run `bash .github/scripts/validate-scope-to-diff.sh` to confirm 15/15 before re-invoking IAA.
