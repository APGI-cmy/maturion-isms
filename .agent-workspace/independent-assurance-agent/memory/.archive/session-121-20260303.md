## Learning Notes

1. **Multi-commit SCOPE_DECLARATION staleness**: When a PR involves multiple sequential commits (original feat → fix → remediation planning → final fix), SCOPE_DECLARATION must be updated to include ALL files modified across ALL commits — not just the files in the final fix commit. The fix in ce01561 correctly reset SCOPE_DECLARATION for the 4 core deliverables but omitted 2 IAA workspace files from 1bb638a. The SCOPE_DECLARATION author must run `git diff --name-only origin/main...HEAD` (or equivalent) after EVERY commit and before IAA invocation to catch this. This is A-026's exact intent.

2. **OVL-CI-005 evidence is not optional for workflow-adding PRs**: When a PR adds a new `.github/workflows/` file, OVL-CI-005 always applies. The evidence must be in the PREHANDOVER. The fact that the workflow may not fire on the PR's own pushes (because trigger conditions aren't met) does NOT exempt the requirement. The workflow DID fire on this PR (run 22634235967 on sha 4f3ee832 — the workflow fires on pull_request:synchronize events for wave/AAWP/MAT PRs, and this PR appears to have met those criteria on the first push). The URL was discoverable but not cited.

3. **Almost-complete remediation**: 10 out of 10 session-120 failures were addressed. 2 new findings emerged — both straightforward. F-011 requires adding a URL to the PREHANDOVER (1-line fix). F-012 requires adding 2 file entries to SCOPE_DECLARATION (2-line fix). Session-122 should be the final invocation if these 2 fixes are committed cleanly.

4. **Substantive quality: STRONG**: The workflow's technical content is excellent — env: variable pattern correctly applied, checkout strategy correct (head.sha for PR events / default branch for issue_comment), reminder comment content accurate, obligation list complete. The REJECTION-PACKAGE is entirely about ceremony completeness (OVL-CI-005 URL missing) and scope declaration completeness (2 files absent). Not a single substantive content concern.

## Suggestions for Improvement (MANDATORY)

1. **Multi-commit SCOPE_DECLARATION audit gate**: A-026 requires exact match between SCOPE_DECLARATION and `git diff --name-only origin/main...HEAD`. For multi-commit fix sequences, the producing agent should run this diff check after EVERY new commit (especially intermediate "planning/remediation" commits) and update SCOPE_DECLARATION before invoking IAA. A concrete addition to the foreman-v2 contract's pre-IAA checklist: "Run `git diff --name-only origin/main...HEAD | sort > /tmp/actual-diff.txt && grep '^\- ' SCOPE_DECLARATION.md | awk '{print $2}' | sort > /tmp/declared.txt && diff /tmp/actual-diff.txt /tmp/declared.txt` before invoking IAA."

2. **OVL-CI-005 explicit checklist item**: The PREHANDOVER template (`.agent-workspace/foreman-v2/knowledge/prehandover-template.md`) should include a specific checkbox for OVL-CI-005: "[ ] OVL-CI-005: CI run URL or log snippet for any `.github/workflows/` files added/modified — URL: ___________". This would prevent the claim-without-URL pattern seen in this session.
