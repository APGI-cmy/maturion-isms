## Learning Notes
1. **Comprehensive coverage for A-029-immutable instances**: When a correction addendum is created for documentation errors in A-029-immutable files, it must cover ALL instances of that error across ALL ceremony artifacts — not just the files explicitly cited in the prior rejection. The Foreman PREHANDOVER (the primary Foreman ceremony artifact) should be the FIRST artifact checked for a documentation error, not discovered as a gap in a re-invocation. Suggest: Foreman's correction addendum should start by listing the error in its own PREHANDOVER before builder PREHANDOVERs.
2. **Mutable session memories must use correct data at creation time**: Foreman session memory is created AFTER the session completes, not DURING it. When the Foreman creates a session memory as part of a STOP-AND-FIX commit (post-error-discovery), all data in that session memory must reflect the CURRENT known-correct state, not the historical QP evaluation state. A session memory created as part of a STOP-AND-FIX is a forward-looking governance artifact, not purely a historical record.
3. **Exhaustive artifact scan is mandatory**: IAA must scan ALL artifacts in the PR bundle for a known error pattern — not just the specific files cited in the prior rejection. The policy-count error was well-known; IAA performed a comprehensive grep and found instances the Foreman missed. This is correct IAA behaviour — exhaustive scanning prevents "whack-a-mole" rejection cycles where only the cited file is fixed each time.
4. **STOP-AND-FIX cycles benefit from a full-artifact audit before commit**: Foreman should run a comprehensive search for ALL instances of a known error before committing a STOP-AND-FIX. A one-liner like `grep -rn "15 policies" .agent-workspace/ .agent-admin/` before committing would have surfaced CORE-021-F1 and CORE-021-F2.
