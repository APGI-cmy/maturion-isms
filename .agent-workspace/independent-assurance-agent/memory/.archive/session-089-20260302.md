## Learning Notes

1. **3-failure remediation pattern is clean**: CodexAdvisor resolved all 3 failures cleanly in the re-submission. This is the ideal STOP-AND-FIX loop behavior — all cited failures resolved, no new failures introduced. The system is working as designed.

2. **SHA256 "before" hash independent verification**: This session verified the "before" hash by running `git show HEAD~1:governance/canon/INDEPENDENT_ASSURANCE_AGENT_CANON.md | sha256sum`. This is the gold standard for OVL-CG-005 "before" hash verification. Future sessions should explicitly run this command when OVL-CG-005 is applicable and a "before" hash is claimed.

3. **grep exit code 1 on zero matches is a bash scripting pitfall**: Encountered in this session during the Merge Gate Parity Check (§4.3) when verifying placeholder hashes in CANON_INVENTORY.json. The command `grep -c 'pattern' file || echo 0` caused `|| echo 0` to execute when grep found 0 matches (exit code 1), producing a double-output `0\n0` that made the comparison `"0\n0" == "0"` fail — generating a false FAIL verdict. Resolved by re-running with `grep -o ... | wc -l`. IAA should always use `grep -o ... | wc -l` or `awk` count for zero-match checks in parity scripts to avoid this false positive pattern.

4. **Post-ASSURANCE-TOKEN ceremony is critical**: The PREHANDOVER proof contains `iaa_audit_token: PENDING` and an empty `## IAA Agent Response (verbatim)` section. CodexAdvisor MUST complete the ceremony: populate the IAA response section verbatim, update the token, commit. This is a conditional pass (CORE-016 mid-ceremony state). If CodexAdvisor opens the PR without completing the ceremony, the `## IAA Agent Response (verbatim)` section will be empty and any subsequent re-audit would fail CORE-016. The ceremony requirement is stated explicitly in the assurance artifact.
