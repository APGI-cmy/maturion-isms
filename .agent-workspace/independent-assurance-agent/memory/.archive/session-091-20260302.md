## Learning Notes

1. **CRITICAL PATTERN — THREE-SESSION SAME-CHECK FAILURE**: This is the THIRD consecutive REJECTION-PACKAGE for PR #789 and the SECOND consecutive failure for OVL-CI-005. The root cause in both OVL-CI-005 failures is identical: the fix exists in the local working tree but has not been committed and pushed to the PR branch. The Foreman prepared the fix correctly but failed to execute the final `git commit && git push` step. This suggests the Foreman's pre-handover checklist needs an explicit "commit verification" step before invoking IAA.

2. **Staged vs. Working Tree Confusion**: The staging area (git index) and working tree are diverged — the staged version has the wrong URL (22577295433), the working tree has the correct URL (22574538846). When committing, the Foreman must ensure the working tree state (not the staged/index state) is what gets committed. Explicit use of `git add <file>` (re-staging from working tree) before committing is required.

3. **CANDIDATE A-021 reinforcement**: The session-090 A-021 candidate ("When CI runs are in `action_required` state, PREHANDOVER must explicitly acknowledge this state") is validated by run 22574538846 being `completed/action_required`. The explicit status explanation in the working tree PREHANDOVER ("action_required — the workflow ran to the point where the Vercel deployment step requires CS2 environment approval") is exactly the right pattern. Elevate A-021 to active rule.
