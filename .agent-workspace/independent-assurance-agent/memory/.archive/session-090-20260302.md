## Learning Notes

1. **GitHub bot PR approval pattern (NEW LEARNING)**: When a PR is created by a bot (Copilot agent), GitHub's "action_required" protection prevents CI workflows from executing until a human (CS2) approves them. This creates a structural gap for OVL-CI-005 compliance: the Foreman cannot obtain a CI run URL without CS2's intervention. **Recommendation**: The Foreman's handover protocol for CI_WORKFLOW category PRs should include an explicit step: "Check if PR CI runs are `action_required`. If so, include a note in PREHANDOVER: 'CI pending CS2 approval — run URL to be added after CS2 approval at [PR URL]'. This is a valid OVL-CI-005 compliance state ONLY if explicitly acknowledged."

   **CRITICAL DISTINCTION**: A Foreman who EXPLICITLY acknowledges "CI pending CS2 approval" in the PREHANDOVER (as the compliance strategy) should be treated differently from a Foreman who either (a) provides no CI evidence or (b) cites the wrong CI run URL. The former is honest process documentation; the latter is incomplete evidence. The current PREHANDOVER has neither acknowledgment in the committed artifact — it simply has no CI Run Evidence section.

2. **Working tree vs. committed artifact ambiguity**: The Foreman added the CI Run Evidence section to the working tree PREHANDOVER but forgot to commit it. For IAA auditing, the committed artifact is the authoritative one (it's what's in the PR). Working tree changes that are not committed are NOT part of the evidence bundle regardless of whether they contain correct content. Future Forem handovers should include a `git status` check as a pre-handover step.

3. **Wrong URL citation pattern**: The working tree PREHANDOVER cited run 22577295433 (a "Copilot coding..." dynamic run) as the CI evidence. This is the IAA invocation session itself, not a deployment workflow run. When adding CI run URLs to PREHANDOVER evidence, the Foreman must ensure the URL corresponds to the specific workflow file that was modified. The `gh run list --workflow=deploy-mat-vercel.yml` command is the correct way to identify the relevant run.
