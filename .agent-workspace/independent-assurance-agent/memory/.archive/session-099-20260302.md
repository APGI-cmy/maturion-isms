1. **Three-invocation cycle convergence pattern**: PR #805 is now on its third REJECTION-PACKAGE. This is a "fix introduces new failure" anti-pattern. Root cause: fixes were applied in the working tree without committing, so the artifacts presented for audit were either wrong or unverifiable via committed state.

2. **Uncommitted artifacts pattern**: This is the third occurrence of A-021 (sessions 090, 091, 099). All three involved governance ceremony artifacts (PREHANDOVER, session memory) being prepared but not committed before IAA invocation. The pattern is consistent across different agent classes (foreman in 090/091, liaison in 099). The pre-invocation checklist step ("run git status — confirm no uncommitted changes") is not being followed systematically.

This is the second occurrence of a partial fix introducing a new failure. The narrative description error (line 257) is independent of the SHA256 correctness.

## Suggestions for Improvement (MANDATORY — this field may NEVER be blank)

1. **Liaison agent pre-commit checklist**: The governance-liaison-isms agent's pre-IAA checklist should include a mandatory step: "Run `git status`. If ANY governance artifact (PREHANDOVER, session memory, HANDOVER_SUMMARY) appears under 'Untracked files' or 'Changes not staged for commit', STOP — commit and push ALL of them before invoking IAA." This one checklist item would have prevented CORE-021-F3 and the equivalent A-021 failures in sessions 090/091.

2. **Grep-all before declaring fix complete**: When a REJECTION-PACKAGE finding involves a text string error (e.g., wrong version number), the fix procedure should include: `grep -rn "[error string]" [artifact file]` to verify ALL instances are corrected before re-invoking IAA. A single missed instance (line 257 out of 5 corrected instances) constitutes a new REJECTION-PACKAGE failure.
