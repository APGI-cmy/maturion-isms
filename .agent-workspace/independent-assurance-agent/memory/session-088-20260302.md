## Suggestions for Improvement

1. **CORE-018 PENDING carve-out needed**: CORE-018's "PREHANDOVER proof file on branch" phrasing should explicitly address working-tree-only artifacts for PENDING ceremonies. Suggested clarification: "PREHANDOVER proof file present and accessible (committed on branch for final merge; working-tree untracked acceptable for PENDING ceremony state with commit-before-merge mandate)." This prevents future ambiguity when auditing PENDING ceremonies. See session-085 parking station suggestion for background.

2. **IAA trigger table — consider adding GOVERNANCE_AUDIT category**: For formal CI/CD assurance audit sessions (like session-090), there is currently no named category. These sessions correctly fall under EXEMPT (no triggering artifacts), but a named GOVERNANCE_AUDIT category with EXEMPT classification would make the classification faster and more explicit. Would prevent any future ambiguity about whether CI/CD audit sessions are triggered.

3. **Post-ceremony commit mandate**: Add an explicit instruction in the Foreman contract's Step 4.3 (or PREHANDOVER template) requiring commit of all artifacts immediately after IAA token is received and PREHANDOVER is updated — before any `report_progress` call. This prevents a recurrence of INC-IAA-SKIP-002 (committed before ceremony) and its inverse (ceremony completed but artifacts not committed before report_progress).
