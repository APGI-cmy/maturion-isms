## Learning Notes

1. **CANDIDATE A-023 — formal evaluation**: Session-093 identified the pattern "update FAIL-ONLY-ONCE.md rules and history entry without updating the `**Version**:` header." This session confirms the root cause is a human omission in multi-change governance commits. The fix is trivially simple (one-line edit) but produces a full REJECTION-PACKAGE cycle. CANDIDATE A-023 rule: "When adding rules to any Tier 2 knowledge file and adding a version history entry, the `**Version**:` header field MUST be updated to match. Header + history must be in sync. Updating only the history without the header = immediate OVL-KG-002 failure." Recommend CS2 ratification. The Foreman's PREHANDOVER pre-commit checklist should include: "For EACH Tier 2 knowledge file you modified: verify (a) `**Version**:` header updated, (b) version history entry added, (c) index.md row updated. All three in sync."

2. **Six-invocation PR pattern**: PR #789 required 6 IAA invocations across one day. Each REJECTION-PACKAGE revealed a different, incremental governance gap discovered while fixing the previous one. This is NOT a governance failure per se — STOP-AND-FIX is working correctly. However, the pattern suggests that the PREHANDOVER pre-commit checklist (currently in the Foreman's template) may benefit from a structured "knowledge governance mini-checklist" for any session that modifies Tier 2 knowledge files. The Foreman implemented the substantive work (15/15 tests GREEN) correctly from the start; all 6 REJECTION-PACKAGEs were process/governance gaps, not implementation quality gaps. This is worth noting for CS2 visibility.

3. **CANDIDATE A-023 addition**: Ready for CS2 ratification. Triggering incident: cae9a53 — rules added, history updated, index updated, header NOT updated. Three out of four checks done correctly; the header was the single miss. The pattern is predictable and preventable.

4. **OVL-KG-002 first-check speed**: The check is now very fast — `grep "^\*\*Version\*\*:" <file>` and compare to index.md claim. Can be done in seconds. Future sessions: run this as first KNOWLEDGE_GOVERNANCE check before any other OVL-KG.
