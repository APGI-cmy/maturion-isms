## FAIL-ONLY-ONCE Updates

**NEW RULE — A-028 candidate**: SCOPE_DECLARATION.md file bullet format must use hyphen (` - `) separator, not em dash (` — `). The `validate-scope-to-diff.sh` BL-027 script uses `grep -E '^\s*-\s+`[^`]+`\s+-\s+'` (primary) and `grep -E '^\s*-\s+[^\s\`].*\.(ext)'` (fallback). Both require hyphen. Em dash is a silent failure — 0 files parsed, CI exits "empty or malformed". The template shows hyphen. Any producing agent using em dash will silently break BL-027.

**Recommendation**: Codify as A-028 in FAIL-ONLY-ONCE.md: "SCOPE_DECLARATION.md file entries must use hyphen (` - `) separator between backtick-quoted path and description. Em dash (` — `) causes BL-027 grep to return 0 matches → 'SCOPE_DECLARATION.md is empty or malformed' on CI."

**Pattern observation (A-026, 7th occurrence)**: This is the third A-026 failure on the SAME PR (sessions 120, 121, 122). The root cause is structural: each fix commit adds new files to the repo without simultaneously adding them to SCOPE_DECLARATION. A further sub-pattern has been identified: format incompatibility. The em dash issue may have been present since 4f3ee83 (the first commit) — it was not caught in session-120 (SCOPE_DECLARATION entirely wrong) or session-121 (content analysed manually, not script-verified). Session-122 is the first to run the actual grep patterns and confirm the format failure.

## Learning Notes

1. **SCOPE_DECLARATION format verification is mandatory**: IAA must always run the actual `validate-scope-to-diff.sh` grep patterns against SCOPE_DECLARATION.md locally (not just count files manually) to catch both format and content failures. Format failures are silent — the file looks correct to human inspection but yields 0 parsed files by the script. From session-123 onwards: run `grep -E '^\s*-\s+`[^`]+`\s+-\s+' SCOPE_DECLARATION.md | wc -l` first and confirm count > 0 before proceeding to content comparison.

2. **Em dash vs hyphen**: Markdown tooling, AI agent outputs, and copy-paste from formatted text often insert em dash (`—`) instead of hyphen (`-`). The template is explicit (hyphen), but the em dash is visually similar and passes human review. BL-027's grep pattern is hyphen-only. IAA must always do the machine verification, not just human verification.

3. **Multi-commit SCOPE_DECLARATION staleness (3rd occurrence on this PR)**: Session-121 learning note #1 noted this pattern. Session-122 confirms: it persists.
