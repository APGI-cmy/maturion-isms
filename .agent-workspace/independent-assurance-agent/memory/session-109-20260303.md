## Learning Notes

1. **Empty-commit invocation pattern**: This session reveals a new failure mode variant: the entire implementation described in the invocation message existed only in the local working tree (or was planned but not yet executed), yet IAA was invoked as if the implementation was committed. The "Initial plan" commit with zero file changes is the tell. IAA must always verify branch state via `git diff --name-only origin/main origin/<branch>` before accepting any invocation's claims about what was changed.

2. **Character count discrepancy as diagnostic**: The claimed character counts in the invocation message (e.g., 29,879 for governance-liaison-isms-agent.md) differ systematically from the branch-actual counts (29,855). This 11–38 char discrepancy across all 9 files is consistent with the parking-station path changes being computed/anticipated but not committed. Character count claims in invocation messages that differ from git-verified counts are a reliable diagnostic for un-committed implementation.

None of these claims constitute evidence. IAA must verify every claim independently against the committed git tree. Rich invocation messages do not compensate for missing committed artifacts.
