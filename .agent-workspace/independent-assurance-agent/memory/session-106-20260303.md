## Learning Notes

1. **Re-invocation without committing is the most common A-021 pattern**: The producing agent applied all 7 session-105 fixes in the working directory and immediately re-invoked IAA without committing and pushing. This is precisely the anti-pattern A-021 was created to prevent. IAA cannot audit working-tree-only changes because those changes are not visible in the PR on GitHub and would not pass CI checks. The fix in the working tree is correct; only the commit step was missing.

2. **PREHANDOVER proof SHA256 hashes must be re-computed after any file modification**: Fix B (adding version history tables) and Fix C (updating index entries) changed the content of index.md, but the PREHANDOVER proof SHA256 hash for index.md was not updated. FAIL-ONLY-ONCE.md and session-memory-template.md hashes were correctly updated, which shows the producing agent understood the requirement — but missed the index.md update. Any time a file is modified as part of a fix, ALL PREHANDOVER proof hashes for affected files must be recomputed.

3. **The iaa_audit_token PENDING text should reference the current session**: The working-tree fix sets `iaa_audit_token: PENDING (IAA session-105 invocation in progress)` but session-105 has concluded with a REJECTION-PACKAGE. The parenthetical text should reference session-106 (the re-invocation) or simply say "PENDING". This is a minor accuracy issue but noted for quality.

4. **Working-tree fix quality was otherwise correct**: Excluding the commit omission and the hash mismatch, all 7 session-105 fixes were applied correctly in the working tree. Fix A (ceremony fields), Fix B (version history tables), Fix C (index versions), Fix D (A-09→A-009) and the reviewer comment fixes were all confirmed correct. Once committed with the pre-commit fix (hash update), the re-invocation should pass.
