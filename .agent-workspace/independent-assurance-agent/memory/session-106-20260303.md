## Learning Notes

1. **Claimed-resolved vs. committed-artifact discrepancy pattern**: OVL-CI-005 was claimed resolved externally ("CI run URL recorded") but the committed PREHANDOVER proof artifact on the branch was never updated with the URL. This is a new failure mode: the producing agent took the external action (PR creation, CI run observed) but did not update the committed artifact file to reflect it. A-021 confirms artifacts must be committed before invoking IAA — but there is a complementary pattern here: the SPECIFIC CONTENT of committed artifacts must reflect claimed resolutions, not just the external actions. The producing agent should verify committed file content (`git show HEAD:path | grep "expected value"`) for every finding claimed as resolved before re-invoking IAA.

2. **Checklist item vs. content discrepancy**: The PREHANDOVER checklist has `[x] CI Run Evidence section populated (OVL-CI-005)` marked as complete, but the section content shows `N/A`. This represents a second failure mode — a checked-off checklist item that does not correspond to complete evidence in the section body. OVL-CI-005 requires a URL or log snippet; `N/A` is not either. Producing agents should validate that checked items are substantively satisfied, not merely that a section header exists.

3. **Near-complete resolution**: 4 of 5 original session-105 findings were verified fully resolved in the committed artifacts. Only OVL-CI-005 remains. This is a strong positive signal for efficiency in the next invocation — a single targeted fix should yield ASSURANCE-TOKEN.
