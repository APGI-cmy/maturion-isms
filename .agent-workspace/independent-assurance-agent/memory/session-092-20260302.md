## Learning Notes

1. **KNOWLEDGE_GOVERNANCE trigger missed for TWO consecutive sessions (090/091)**: When commit dfdc6ba added the foreman-v2 FAIL-ONLY-ONCE.md update to PR #789, sessions 090 and 091 classified the PR as MIXED (CI_WORKFLOW + AAWP_MAT) without recognising the KNOWLEDGE_GOVERNANCE trigger. The trigger table unambiguously covers `.agent-workspace/*/knowledge/*` files. **CANDIDATE A-022**: "When a PR receives new commits between IAA invocations, IAA must re-evaluate ALL trigger categories against the full commit history, not just the categories identified in the first invocation. New commits may introduce new trigger categories that were absent in earlier versions of the PR. IAA must re-read the trigger table for every session, not carry forward the prior session's category classification."

2. **OVL-KG-003 format mismatch**: The OVL-KG-003 check ("version history table updated") was designed for IAA Tier 2 knowledge files which always have `## Version History` tables. The foreman-v2-agent FAIL-ONLY-ONCE.md uses a different format that pre-dates OVL-KG-003. **CANDIDATE A-022 addendum**: Add OVL-KG-003 carve-out note: "If the modified file does not use the IAA knowledge format (no `## Version History` table), IAA must verify that version changes are documented through the file's own versioning mechanism (header version + change documentation). N/A is permitted with explicit justification. The producing agent should add a `## Version History` table in a future governance PR to bring the file into full OVL-KG-003 compliance."

3. **OVL-CI-005 FULLY RESOLVED**: The fix in d27f876 is correct. Run 22574538846 is the right run (deploy-mat-vercel.yml, PR #789, completed/action_required).
