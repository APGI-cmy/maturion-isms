## Learning Notes

1. **New workflow addition vs existing workflow modification for OVL-CI-005**: This session confirms that OVL-CI-005 applies to NEW workflow additions, not just modifications. For NEW workflows, the technical constraint is that the workflow itself cannot self-trigger on a PR (only base-branch workflows run on PRs). The required evidence path for a new workflow is: (a) get CS2/maintainer to approve GitHub Actions runs, (b) cite the resulting run URLs (from other workflows executing on the PR, or from a manual test run). Local YAML validation is NOT sufficient. Recommend adding this carve-out explicitly to OVL-CI-005 in a future overlay update.

2. **`action_required` conclusion pattern**: When Copilot bot PRs are submitted, GitHub Actions runs may go into `action_required` state requiring maintainer approval before executing. This creates a catch-22: governance workflows need to run to prove they work, but they need approval to run. The Foreman should flag this in the PREHANDOVER and coordinate with CS2 for workflow approval before IAA invocation. This pattern should be added to the PREHANDOVER template as a "CI approval coordination" step for Copilot-authored PRs.

3. **POLC boundary gate push-event validation runs**: Two runs (22585179143, 22585724029) showed `conclusion: failure` for `.github/workflows/polc-boundary-gate.yml` on push events, with `total_jobs: 0`. These appear to be GitHub's internal workflow-file validation runs (not the actual polc-boundary-gate running). The PREHANDOVER should acknowledge these and either explain them or confirm they are expected validation artifacts (not actual gate failures). Including CI run context in the PREHANDOVER provides clearer evidence.

4. **OVL-CI-006 is consistently missed**: Both this session (PR #814) and sessions 088–094 (PR #789) saw OVL-CI-006 surface. The PREHANDOVER template may not have a dedicated environment parity section. Confirm A-020 check: if template still lacks an `## Environment Parity` section, the template needs an update before the next wave.
