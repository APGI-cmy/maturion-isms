## Learning Notes

1. **O-001 mid-session fix pattern**: The producing agent applied a CS2 observation fix (O-001 directory guard) partway through the session. This changed the SHA256 of `agent-contract-audit.yml`. The agent updated the HANDOVER_SUMMARY with a placeholder `—` acknowledging the staleness, but did NOT update the session memory, and DID commit both artifacts with stale/incomplete hashes. This "mid-session fix without hash cascade" pattern should be flagged. It causes both CORE-007 and CORE-020 failures simultaneously.

   **Pattern name**: "Post-O-fix hash cascade miss" — when a fix is applied after evidence artifacts are written, ALL hash references in ALL committed artifacts must be updated atomically before committing.

2. **OVL-CI-005 zero CI runs**: The PR had 0 CI check runs at the time of IAA invocation. This is the second time OVL-CI-005 has appeared as a failure (first was sessions 090-092 for PR #789). The PREHANDOVER template (in Foreman's memory space) does not enforce the `## CI Check Run Evidence` section for CI_WORKFLOW PRs. A-020 applies: the Foreman/governance-liaison PREHANDOVER template lacks the required CI_WORKFLOW overlay sections.

3. **OVL-CI-006 missing environment parity**: Simple to fix with a one-paragraph statement. The PREHANDOVER template should include this section by default for any CI_WORKFLOW PR.

4. **PREHANDOVER location in Foreman memory**: The governance-liaison-isms agent placed the PREHANDOVER in `.agent-workspace/foreman-v2/memory/` rather than its own workspace. This is not a failure per current checklist but is worth noting — the governance-liaison-isms agent appears to be using the Foreman's PREHANDOVER template format. If the Foreman's template is stale (per A-020), the governance-liaison's PREHANDOVER will also be stale.

## Suggestions for Improvement (MANDATORY)

The governance-liaison-isms PREHANDOVER template (shared with or derived from the Foreman's template) does not include `## CI Check Run Evidence` or `## Environment Parity` sections, which are required by the CI_WORKFLOW overlay (OVL-CI-005 and OVL-CI-006). This is the third case (sessions 088-093 for PR #789, and now this session for PR #812) where OVL-CI-005 has produced a REJECTION-PACKAGE on a first invocation. **Concrete improvement**: The governance-liaison-isms PREHANDOVER template must be updated to include both `## CI Check Run Evidence` and `## Environment Parity` sections as mandatory fields for any CI_WORKFLOW PR. Additionally, when a mid-session fix changes a committed file's content, ALL SHA256 references across ALL evidence artifacts must be updated atomically — a pre-commit checklist item should verify hash consistency across HANDOVER_SUMMARY, session memory, and PREHANDOVER proof before any `git commit`.
