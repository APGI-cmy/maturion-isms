## Learning Notes

**learning_notes**:

1. **Committed merge conflict markers pattern (NEW)**: Session-040 committed `.agent-workspace/parking-station/suggestions-log.md` with unresolved merge conflict markers. This is a new pattern — a file from a multi-branch parallel editing scenario (session-100 IAA commit and session-040 liaison commit both modified suggestions-log.md independently, creating a conflict). Agents committing to files that are frequently modified by other agents on concurrent branches must be vigilant about merge conflicts. Recommend adding to all agents' FAIL-ONLY-ONCE: "Verify no merge conflict markers exist in committed files before invoking IAA."

2. **SHA256 computation timing invariant**: The PREHANDOVER SHA256 table must be computed AFTER all artifacts are finalized, not before. Session-040's SHA256 table was written during PREHANDOVER creation, then the session memory was amended (adding IAA invocation result, parking station entry), invalidating the hash. Rule: SHA256 computation is the LAST step before IAA invocation, not an intermediate step.

Even when re-auditing a PR, new trigger categories can emerge from incremental commits.
