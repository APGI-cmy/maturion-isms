## Learning Notes

1. **A-022 critical finding**: The fix commit (795e030) itself introduced a new trigger category (KNOWLEDGE_GOVERNANCE) by modifying FAIL-ONLY-ONCE.md. The session-101 category classification (AGENT_CONTRACT only) was no longer accurate after the fix commit was applied. A-022 correctly caught this. This demonstrates exactly why A-022 exists — remediation commits can introduce new trigger categories.

2. **KNOWLEDGE_GOVERNANCE ceremony gap**: When CodexAdvisor (or any agent) modifies an IAA Tier 2 knowledge file (FAIL-ONLY-ONCE.md, iaa-core-invariants-checklist.md, etc.) as part of a session bundle commit, the KNOWLEDGE_GOVERNANCE overlay checks MUST be satisfied: version bump, version history update, index.md update. The PREHANDOVER must cover KNOWLEDGE_GOVERNANCE changes, not just AGENT_CONTRACT changes. In this PR, the PREHANDOVER only addressed AGENT_CONTRACT changes and made no mention of the FAIL-ONLY-ONCE.md modification.

3. **CORE-009 systemic builder gap**: The pre-existing gap in api-builder.md and qa-builder.md (parity_required/parity_enforcement absent) was not caught by session-101. This suggests IAA may need a dedicated CORE-009 check in its pre-check pass for each agent contract file in the diff. Consider adding to PREHANDOVER template checklist.

4. **Incremental fix commits require full re-audit**: When CodexAdvisor prepares a "fix commit" addressing a prior REJECTION-PACKAGE, the fix commit itself may introduce new governance failures (as happened here). IAA must treat every new commit on the branch as potentially adding new trigger categories or new CORE violations.

5. **FAIL-ONLY-ONCE.md modification authority**: Per "Adding New Rules" section in FAIL-ONLY-ONCE.md, "IAA adds a new entry to this file... All updates to this file must be committed as part of the session bundle for that invocation." CodexAdvisor committed A-023 to IAA's FAIL-ONLY-ONCE.md on IAA's behalf (since session-101 was committed together with the fix bundle). While CodexAdvisor has write_paths to .agent-workspace/ directories, the KNOWLEDGE_GOVERNANCE versioning obligations were not fulfilled. Future recommendation: when IAA's session memory (fail_only_once_updates) says a new rule was added, CodexAdvisor must ALSO bump the FAIL-ONLY-ONCE.md version and update index.md as part of the same commit.
