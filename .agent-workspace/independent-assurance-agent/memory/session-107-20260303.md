## Learning Notes

1. **SHA256 hash staleness is a persistent pattern across three consecutive sessions (106, 107)**: Session-106 found index.md hash stale after Fix B/C modifications. Session-107 found 11 of 16 declared hashes stale after fix commit modified parking station files, governance spec, and session-039 memory. The producing agent correctly updated Tier 2 knowledge file hashes but systematically failed to update hashes for the other modified files. This is the most recurring post-fix re-invocation failure pattern in this PR lifecycle. Pattern requires a new FAIL-ONLY-ONCE rule in IAA's registry.

2. **Fix commit scope vs. PREHANDOVER proof update scope must be identical**: When a fix commit modifies files F1, F2, ..., Fn, the PREHANDOVER proof must be updated for all of F1, F2, ..., Fn that have declared SHA256 hashes. The fix commit did not document which files it would modify, leading to partial hash updates. Producing agents must systematically apply `sha256sum` to ALL files listed in the PREHANDOVER proof after any modification, not just the files they believe they changed most significantly.

3. **A-011 gap was a session-105 fix requirement that was partially carried through sessions 106 and 107**: Session-105 OVL-KG-005 cited both (a) A-09 padding AND (b) A-011 gap. Session-106 cited OVL-KG-005 specifically for the A-021 violation (uncommitted state). Session-107: Fix D (padding) is now committed, but Fix E (A-011 gap) was never executed. This is a case where a multi-component fix requirement had only its most-cited component addressed. Producing agents must address ALL sub-components of a fix requirement, not just the primary one cited in re-invocation.

4. **Positive progress is substantial**: Sessions 105 and 106 had 7 and 10 failures respectively. Session-107 has only 2 failures. All structural issues from sessions 105 and 106 are resolved: iaa_audit_token PENDING (PASS), IAA Agent Response section (PASS), version history tables all 3 files (PASS), index.md updated (PASS), A-09→A-009 committed (PASS), A-021 compliance (PASS). Only the SHA256 update completeness and A-011 gap remain. Session-108 should succeed if both are addressed.

5. **governance-liaison-isms parking station file hash mismatch may predate fix commit**: The governance-liaison-isms parking station file was NOT in the fix commit stat for 6f145a0, yet its declared hash doesn't match. This suggests the hash was incorrect in the original PREHANDOVER proof. This may indicate a systematic hash computation error in the original commit's PREHANDOVER proof for some files. Producing agents should verify ALL hashes against the ACTUAL committed files, not just files they believe they modified.
