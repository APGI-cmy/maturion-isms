# IAA — ECAP Three-Role Split Checklist

Use this checklist whenever a wave appoints `execution-ceremony-admin-agent`.

## Independence checks

- Foreman invoked IAA, not ceremony-admin
- IAA did not prepare or modify the ceremony bundle
- ceremony-admin did not issue a token or rejection package

## Bundle checks

- PREHANDOVER proof exists and is committed
- session memory exists and is committed
- Foreman reviewed the returned bundle before IAA invocation
- token file is a dedicated append-only artifact

## Role-boundary checks

- Foreman retained substantive readiness judgment
- ceremony-admin performed administration only
- IAA performed assurance only
- no contract or PR text implies substitution between the three roles

## Failure language to reject

- "ceremony-admin invoked IAA"
- "ceremony-admin approved readiness"
- "IAA completed cleanup"
- "Foreman no longer accountable once ceremony-admin appointed"
