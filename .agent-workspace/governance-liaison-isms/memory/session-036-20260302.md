## Suggestions for Improvement (MANDATORY)

The `polc-boundary-gate.yml` was running on push events despite only declaring `pull_request` as a trigger — this is a GitHub Actions quirk where workflow runs are created for push events under certain conditions (e.g., during PR merge evaluation). Future improvement: consider adding a `push: branches-ignore: ['**']` (no-op push block) to the workflow trigger explicitly, so the intended-only-pull_request nature is self-documenting and GitHub's behavior is unambiguous. The `if:` guard is the authoritative fix, but explicit trigger exclusion would add clarity.
