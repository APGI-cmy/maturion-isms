### CI Failure Fix

Root cause: `peter-evans/create-pull-request@v6` fails the step when label application
encounters a stale GraphQL node ID. GitHub Actions skips subsequent steps (including
`Resolve PR number`) when a preceding step fails, causing `PR_NUMBER=""` in the comment
step, which triggers the "Alignment Script Error" branch.

Fix applied: Added `continue-on-error: true` to both `create_pr_standard` and
`create_pr_draft` steps in `.github/workflows/ripple-integration.yml`. This ensures:
1. Label application failures do not fail the overall step
2. `pull-request-number` output remains available for subsequent steps
3. PR number is resolved correctly
4. Comment shows correct status message
