### What Future Sessions Should Know
- When adding devDependencies with a version range like `^20.0.0`, the pnpm-lock.yaml resolution
  or a merge from main may update the declared version. Always verify the actual package.json
  content on the branch rather than relying on what you declared in a prior commit.
- HANDOVER_SUMMARY "Files Modified" table must only list files that actually exist. Do not claim
  files as CREATED until they are actually committed.
- The PREHANDOVER proof must be created BEFORE the first IAA invocation, not after the
  first REJECTION-PACKAGE. The ceremony order is: fix → validate → create PREHANDOVER → invoke IAA.
