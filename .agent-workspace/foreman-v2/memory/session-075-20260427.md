## Wave Summary

**Wave type**: CI_WORKFLOW hardening (deployment workflow fix)
**Purpose**: Fix the failing `Deploy MMM Supabase Migrations` workflow by replacing the interactive `supabase link + supabase db push` mechanism (which failed with SASL auth errors on GitHub-hosted runners) with the proven non-interactive Management API approach (`apply-migrations-via-api.py`).

**Root cause of prior failure**: `supabase db push` prompts for DB password interactively when the linked project lacks the credential. GitHub-hosted runners cannot authenticate non-interactively against the Supabase pooler without explicit DB password provision.

**Approved fix**: Replace with `python3 .github/scripts/apply-migrations-via-api.py --migrations-dir supabase/migrations/ --tracking-table mmm_native_migrations` — identical pattern to the already-proven cross-app (legacy/AIMC) migration steps.

**Also fixed**: Removed `supabase/setup-cli@v1` step (Supabase CLI no longer needed). Fixed stale documentation comments referencing `supabase db push` as the approved mechanism.
