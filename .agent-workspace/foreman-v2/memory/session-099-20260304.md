## Suggestions for Improvement

**Concrete improvement from this session**: SCOPE_DECLARATION.md format requirement (` - ` vs `—` separators)
is not documented in the foreman PREHANDOVER template or in any Tier 2 builder-facing instructions.
The IAA session-135 REJECTION-PACKAGE was caused by this undocumented format constraint. Recommend
adding a SCOPE_DECLARATION format note to the Tier 2 prehandover-template.md with an explicit
example showing the ` - ` separator format and a reminder to run `validate-scope-to-diff.sh` locally
before IAA invocation. Additionally, the validate script should handle shallow-clone environments
(where origin/main ref is absent) by falling back to a documented bypass path, rather than silently
returning 0 changed files and causing spurious mismatch errors.
