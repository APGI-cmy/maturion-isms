## What Future Sessions Should Know

1. **Python3 JSON parse pattern** is preferred for package.json scanning. More reliable
   than grep/jq for structured JSON with nested keys (dependencies vs devDependencies).
2. **@langchain/* wildcard**: Use `startswith('@langchain/')` — covers all scoped packages.
3. **RED state proof without test runner**: When vitest is unavailable, RED state can be
   proven through logical argument: if the file under test doesn't exist, `fs.existsSync()`
   returns false, and `expect(false).toBe(true)` fails. Document this in PREHANDOVER.
4. **S-033 OVL-CI-005**: New CI workflows always trigger S-033 exception. Document all
   3 conditions (YAML lint, pattern parity, workflow_dispatch) proactively.
5. **A-031 carve-out**: Always include A-031 carve-out section in SCOPE_DECLARATION when
   IAA ceremony artifacts may appear in the diff. Prevents HFMC-02 false failures.
