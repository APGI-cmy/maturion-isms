### What Future Sessions Should Know

1. **STOP_AND_FIX_DOCTRINE.md already covers lint** - Line 57, line 322, line 867 all mention linter errors as zero-tolerance items. Our update strengthens enforcement in handover templates but doesn't create new doctrine.

2. **PREHANDOVER_PROOF_TEMPLATE.md v3.0 structure** - Template uses numbered gates (Gate 1, Gate 2, etc.) with consistent format. When adding new gates, maintain this structure and renumber subsequent gates.

3. **Wave Completion Gate vs Wave Closure Certification** - These are distinct concepts in FULLY_FUNCTIONAL_DELIVERY_STANDARD.md:
   - Wave Completion Gate: Pre-closure validation (Section 4.2)
   - Wave Closure Certification: Formal certification after validation (Section 4.3)

4. **Foreman authority for governance updates** - This type of governance documentation update is within Foreman authority. No builder needed, no escalation required. Foreman can self-execute governance enhancements that strengthen existing constitutional requirements.

5. **Evidence documents location** - Created at repository root level (not in .agent-workspace) because they are deliverables for human review, not just agent memory

6. **placeholder scripts are intentional** - Don't assume placeholder scripts need to be replaced with real implementations. They serve as gate existence validators and allow each project to choose their own linting/type-checking tools.
